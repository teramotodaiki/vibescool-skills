# 評価項目（固定）

`session-retrospective` では、以下5項目を毎回同じ順番で評価する。
点数は **1-5 点**（5が最も良い）。

1. 課題理解と計画性 (`goal_planning`)
- 依頼内容の把握、目的の確認、作業順序の整理ができたか。

2. 実装の実行力 (`execution`)
- 実際の編集・コマンド実行・検証までやり切れたか。

3. 問題解決力 (`debugging`)
- エラー時に原因を特定し、再現確認と修正を行えたか。

4. 対話と伝達力 (`communication`)
- 依頼者要望を正確に反映し、進捗や判断を明確に伝えられたか。

5. 振り返りと改善意識 (`reflection`)
- 制約や失敗を次の改善に繋げる行動が見られたか。

## 採点ルール

- 各項目に必ず短いコメントを付ける。
- 1回のレポートで5項目すべてを採点する。
- 点数とコメントは、必ずセッションログの根拠（line番号）と対応させる。
